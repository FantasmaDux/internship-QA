# Список тест-кейсов для микросервиса

Содержит E2E тест-кейсы для проверки микросервиса Авито. Микросервис содержит 4 эндпоинта:
- Создать объявления
- Получить объявления по его идентификатору
- Получить все объявления по идентификатору продавца
- Получить статистику по айтем ID

Host: https://qa-internship.avito.com

**Окружение:** Postman Version 11.85.0

**Общие предусловия:**
- импортировать Postman-коллекцию запросов avito-qa-internship.json

**Приоритеты проверок**:
- Высокий. Если функциональность не работает, найден критический баг. Соответствует основной позитивной проверке.
- Средний. Важная часть логики эндпоинта.
- Низкий. Второстепенная и редко используемая логика эндпоинта.

**Правила именования идентификаторов тест-кейсов:**
Пример: TC_E2E_01

## Применяемые техники тест-дизайна:
- Эквивалентное разбиение
- Граничные значения

## Общая статистика по тест-кейсам

| Номер | Название                                                     | Задействованные эндпоинты                      |
|-------|--------------------------------------------------------------|------------------------------------------------|
| 1     | Создание объявления и получение по ID                        | POST `api/1/item`, GET `api/1/item/{id}`       |
| 2     | Создание объявления и получение в списке продавца            | POST `api/1/item`, GET `api/1/{sellerId}/item` |
| 3     | Создание нескольких объявлений и получение в списке продавца | POST `api/1/item`, GET `api/1/{sellerId}/item` |
| 4     | Попытка получения статистики несуществующего объявления      | POST `api/1/item`, GET `api/1/statistic/{id}`  |

**Всего тест-кейсов написано:** 4

## E2E тест-кейсы

### Позитивные тесты

---
#### **Тест-кейс создания объявления с валидными параметрами и его получения по ID**

**Идентификатор**: TC_E2E_01

**Приоритет**: высокий

**Описание**: проверка стандартной последовательности действий при работе с корректно созданным объявлением

**Тестовые данные**:
```
{
  "sellerID": Integer,
  "name": String,
  "price": Long,
  "statistics": {
    "likes": Long,
    "viewCount": Long,
    "contacts": Long
  }
}
```

**Предусловия**: - 

| Номер шага | Шаг                                                                        | Ожидаемый результат                                             |
|------------|----------------------------------------------------------------------------|-----------------------------------------------------------------|
| 1          | Отправить POST запрос создания объявления                                  | Получен код 200 OK                                              |
| 2          | Проверить тело ответа                                                      | Тело ответа содержит идентификатор созданного объявления        |
| 3          | Извлечь идентификатор объявления и указать в GET запросе `api/1/item/{id}` | Получен код 200 ОК                                              |
| 4          | Проверить тело ответа                                                      | Тело ответа содержит полную корректную информацию об объявлении |

**Статус**: пройдено

**Приложение**: -

---

#### **Тест-кейс создания объявления и его получения в списке среди объявлений продавца**

**Идентификатор**: TC_E2E_02

**Приоритет**: высокий

**Описание**: проверка, что при создании объявления с валидными параметрами оно будет добавлено в список объявлений указанного продавца

**Тестовые данные**:
```
{
  "sellerID": Integer,
  "name": String,
  "price": Long,
  "statistics": {
    "likes": Long,
    "viewCount": Long,
    "contacts": Long
  }
}
```

**Предусловия**: - 

| Номер шага | Шаг                                                           | Ожидаемый результат                                      |
|------------|---------------------------------------------------------------|----------------------------------------------------------|
| 1          | Отправить POST запрос создания объявления                     | Получен код 200 OK                                       |
| 2          | Проверить тело ответа                                         | Тело ответа содержит идентификатор созданного объявления |
| 3          | Указать тот же sellerId в GET запросе `api/1/{sellerId}/item` | Получен код 200 ОК                                       |
| 4          | Проверить тело ответа                                         | Тело ответа содержит созданное объявление                |

**Статус**: пройдено

**Приложение**: -

---

#### **Тест-кейс создания нескольких объявлений одним продавцом и проверка их наличия в списке**

**Идентификатор**: TC_E2E_03

**Приоритет**: средний

**Описание**: проверка, что при создании нескольких объявления с валидными параметрами все они будут добавлены в список объявлений указанного продавца

**Тестовые данные**:
```
{
  "sellerID": Integer,
  "name": String,
  "price": Long,
  "statistics": {
    "likes": Long,
    "viewCount": Long,
    "contacts": Long
  }
}
```

**Предусловия**: - 

| Номер шага | Шаг                                                           | Ожидаемый результат                             |
|------------|---------------------------------------------------------------|-------------------------------------------------|
| 1          | Отправить POST запрос создания объявления 1                   | Получен код 200 OK                              |
| 2          | Отправить POST запрос создания объявления 2                   | Получен код 200 OK                              |
| 3          | Отправить POST запрос создания объявления 3                   | Получен код 200 OK                              |
| 4          | Указать тот же sellerId в GET запросе `api/1/{sellerId}/item` | Получен код 200 ОК                              |
| 5          | Проверить тело ответа                                         | Тело ответа содержит все 3 созданных объявления |

**Статус**: пройдено

**Приложение**: -

#### **Тест-кейс попытки получения статистики несуществующего объявления после успешного создания объявления**

**Идентификатор**: TC_E2E_04

**Приоритет**: средний

**Описание**: проверка, что при замене одного символа в UUID успешно созданного объявления и попытке получения статистики с таким ID приходит 404 Not Found

**Тестовые данные**:
```
{
  "sellerID": Integer,
  "name": String,
  "price": Long,
  "statistics": {
    "likes": Long,
    "viewCount": Long,
    "contacts": Long
  }
}
```

**Предусловия**: - 

| Номер шага | Шаг                                                                                    | Ожидаемый результат                                      |
|------------|----------------------------------------------------------------------------------------|----------------------------------------------------------|
| 1          | Отправить POST запрос создания объявления                                              | Получен код 200 OK                                       |
| 2          | Проверить тело ответа                                                                  | Тело ответа содержит идентификатор созданного объявления |
| 3          | Заменить один символ в ID объявления и указать ID в GET запросе `api/1/statistic/{id}` | Получен код 404 Not Found                                |

**Статус**: пройдено
